Budget Optimization Using Data Server

Model Solution — Data Server and Optimization Exercise

CRB(y)0.015 + 0.00015yConversion rate, Product B

1
Total Conversions as a Function of One Variable
8 marks

The expected conversions from each product are the product of the advertising budget and its conversion rate:

Conversions from Product A:
CA(x) = x · CRA(x) = x(0.02 + 0.0001x)
          = 0.02x + 0.0001x²
Conversions from Product B:
CB(y) = y · CRB(y) = y(0.015 + 0.00015y)
          = 0.015y + 0.00015y²
Step 1
Apply the budget constraint: express y in terms of x.
x + y = 10,000  ⟹  y = 10,000 − x
Step 2
Write T(x) = CA(x) + CB(y) and substitute y = 10,000 − x.
T(x) = (0.02x + 0.0001x²) + CB(10,000 − x)
Step 3
Expand CB(10,000 − x):
= 0.015(10,000 − x) + 0.00015(10,000 − x)²
= 150 − 0.015x + 0.00015(100,000,000 − 20,000x + x²)
= 150 − 0.015x + 15 − 3x + 0.00015x²
= 165 − 3.015x + 0.00015x²
Step 4
Combine both terms:
T(x) = (0.02x + 0.0001x²) + (165 − 3.015x + 0.00015x²)
       = (0.0001 + 0.00015)x² + (0.02 − 3.015)x + 165

T(x) = 0.00025x² − 2.995x + 165   (0 ≤ x ≤ 10,000)
✓ Result — Task 1
T(x) = 0.00025x² − 2.995x + 165   [total conversions]
Domain: 0 ≤ x ≤ 10,000 ;  y = 10,000 − x
2
Budget Allocation that Maximizes Total Conversions
8 marks

We apply calculus: find the critical point via T′(x) = 0, then use the second derivative test and evaluate endpoints.

Step 1
Differentiate T(x):
T(x) = 0.00025x² − 2.995x + 165
T′(x) = 2(0.00025)x − 2.995
T′(x) = 0.0005x − 2.995
Step 2
Set T′(x) = 0 and solve:
0.0005x − 2.995 = 0
x = 2.995 ÷ 0.0005
x* = 5,990
Step 3
Second derivative test:
T″(x) = 0.0005 > 0  ⟹  concave up  ⟹  x = 5,990 is a MINIMUM
⚠ Key Observation

T(x) is an upward-opening parabola (positive leading coefficient 0.00025 > 0). On the closed interval [0, 10,000], the maximum must occur at an endpoint. The critical point x = 5,990 is the global minimum.

Step 4
Evaluate T(x) at the endpoints:
T(0) = 0 − 0 + 165 = 165.00

T(10,000) = 0.00025(10,000)² − 2.995(10,000) + 165
             = 25,000 − 29,950 + 165 = −4,785

T(5,990) = 0.00025(5,990)² − 2.995(5,990) + 165 ≈ −8,795 (minimum)
Step 5
Compare endpoints; select the maximum:
T(0) = 165 > T(10,000) = −4,785  ⟹  Maximum at x = 0
✓ Result — Task 2: Optimal Budget Allocation
x* = $0  → Product A   |   y* = $10,000  → Product B
Maximum Total Conversions:  T(0) = 165
📐 Calculus Summary

T′(x) = 0 at x = 5,990 → interior minimum (T″ > 0).
Closed interval [0, 10,000]: compare T(0) = 165 vs T(10,000) = −4,785.
Global maximum: x = 0, T = 165.

3
Data-Driven Interpretation
4 marks

The data server's historical conversion models shaped every step of the optimization. Here is how:

  • Foundation for model construction: CRA(x) and CRB(y) were retrieved from the server's fitted historical records. These linear models (base rate + marginal rate × spend) are empirical — without the data server, there is no T(x) to optimize.
  • Product B has a superior marginal conversion rate: The server reveals CRB has a marginal coefficient of 0.00015 vs. 0.0001 for CRA — each additional dollar spent on B generates 50% more incremental conversions than on A. This data difference directly drives the optimal allocation toward B.
  • Sensitivity to data quality: The result — allocate the full $10,000 to Product B — is entirely contingent on the data server's figures. If historical data were inaccurate or outdated, the optimization output would be invalid, underscoring the importance of clean, reliable server data.
  • Scalability with real-time data: A live data server could supply updated CR coefficients dynamically. The calculus framework remains valid; only the coefficients change. The company could re-run optimization automatically each time the server refreshes conversion metrics, enabling continuous budget reallocation.
✓ Final Conclusion
Allocate $0 to Product A and $10,000 to Product B.
This yields the maximum total conversions: T = 165.
The data server was indispensable — it provided the empirical conversion rate functions that made rigorous calculus-based optimization possible and actionable.
